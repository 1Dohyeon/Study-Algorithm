# 하노이 탑 알고리즘

이 프로젝트는 하노이 탑 문제를 해결하는 Java 프로그램을 제공합니다. 하노이 탑은 다음과 같은 문제입니다.

## 문제 설명

주어진 원반의 개수에 따라 3개의 기둥 사이에서 원반을 옮기는 과정을 나타내는 알고리즘입니다. 주어진 기둥 중 하나는 출발 기둥, 다른 하나는 도착 기둥, 나머지 하나는 임시 기둥으로 사용됩니다.

## 하노이 탑 알고리즘 원리

하노이 탑 알고리즘은 재귀적인 방법을 사용하여 원반을 이동하는 과정을 설명합니다. 다음은 알고리즘의 원리입니다.

1. 원반은 세 개의 기둥 중 하나에서 다른 기둥으로 옮겨져야 합니다. 이때, 출발 기둥, 도착 기둥, 그리고 임시 기둥이 사용됩니다.

2. 원반을 옮길 때는 다음 규칙을 따릅니다.
    - 한 번에 하나의 원반만 옮길 수 있습니다.
    - 큰 원반은 작은 원반 위에 놓을 수 없습니다.

3. 원반의 개수가 1개일 때는 직접 옮길 수 있으므로, 이동 경로를 출력하고 종료합니다.

4. 원반의 개수가 2개 이상일 때, 다음과 같이 재귀적으로 알고리즘을 수행합니다.
    - 가장 큰 원반을 제외한 나머지 원반들을 임시 기둥으로 옮깁니다. 이때, 도착 기둥을 임시 기둥으로 사용합니다.
    - 가장 큰 원반을 도착 기둥으로 옮깁니다.
    - 임시 기둥에 있는 나머지 원반들을 도착 기둥으로 옮깁니다. 이때, 출발 기둥을 임시 기둥으로 사용합니다.

5. 위의 단계를 반복하여 모든 원반을 출발 기둥에서 도착 기둥으로 이동시킵니다.

이러한 과정을 통해 하노이 탑 알고리즘은 주어진 원반 개수에 관계없이 원반을 옮기는 최적의 경로를 찾아냅니다.

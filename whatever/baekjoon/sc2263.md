포스트오더(후위순회)를 하면 루트 노드가 맨 마지막에 나온다는 특징이 있다.
또한 인오더(중위순회)를 한다면 루트노드 왼쪽에는 왼쪽 서브트리,
오른쪽에는 오른쪽 서브트리의 값들이 존재하게 된다.

예를 들어 포스트오더의 결과가 4 5 2 3 1 이고,
인오더의 결과가 4 2 5 1 3 일때, 포스트오더의 마지막 원소인 1은 루트노드임을 알 수 있고,
인오더 결과를 통해 4 2 5는 1의 왼쪽 서브트리, 3은 1의 오른쪽 서브트리임을 알 수 있다.

반복해서 왼쪽 서브트리의 포스터오더가 4 5 2 임으로 루트는 2이고, 4는 왼쪽,
5는 오른쪽 리프노드임을 알 수 있다.

프리오더(전위순회)를 알기 위해서는 루트부터 시작해서 왼쪽을 먼저 탐색하고,
오른쪽을 탐색하면 된다.

main
{

    sb: 결과를 출력하기 위해 Stringbuffer 를 이용(String을 이용하면 메모리 초과)
    inorder: 입력받은 중위순회 결과를 int타입 배열로 저장
    index: inorder 원소의 index값을 저장
    postorder: 입력받은 후위순회 결과를 int타입 배열로 저장

    PreOrder메서드를 이용하여 static으로 선언된 sb(전위순회 결과)를 업데이트 하고, 
    출력한다.
}

PreOrder
(inorder 시작인덱스, inorder 끝인덱스, postorder 시작인덱스, postorder 끝인덱스)
형태로 매개변수를 받는다.
{

    if(inStart > inEnd || postStart > postEnd)
    {
        각 순회의 시작 노드가 끝 노드보다 크다면, return; 
    }

    rootNode: 후위순회의 끝 인덱스 값
    rootNode는 변수명처럼 서브트리의 루트노드이므로 sb에 추가해준다.

    rootIndex: index를 저장해둔 변수 index의 rootNode번째 원소의 값으로,
    inorder를 했을때 루트 노드의 인덱스 값을 가지고 있다.

    즉, 이제 왼쪽 오르쪽 서브트리를 알아냈으므로, 왼쪽 서브트리의 크기를 구해서
    변수 leftSize에 저장한다.

    leftSize 변수를 이용하여 오른쪽 서브트리의 시작 범위를 구하고,
    이 변수를 이용하여 왼쪽 서브트리와 오른쪽 서브트리의 후위순회 범위를 지정한다.

    그 범위들을 매개변수로 하여 왼쪽 서브트리를 먼저 PreOrder 재귀호출을 하고,
    오른쪽 서브트리를 PreOrder 재귀호츨을 하여 값을 구한다.
